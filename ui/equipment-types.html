TYPES D'ÉQUIPEMENTS

⚙️ Types d'équipements — Guide

Cette page configure le comportement du moteur de corrélation CVE pour chaque catégorie d'équipement. Chaque ligne définit comment le moteur recherche les vulnérabilités applicables à un asset.

Colonnes de version

  • OS (FK) — Utilise l'OS normalisé NVD (os_version_id) pour matcher les CVE. Recommandé si l'OS est sélectionné depuis le référentiel.
  • Version OS — Utilise le texte libre version_os pour affiner la comparaison. Désactiver pour les OS dont les builds ne sont pas comparables (ex: Windows Server).
  • Firmware — Utilise le firmware normalisé NVD (fw_version_id) pour les équipements avec firmware distinct (ex: switches, pare-feux).
  • BIOS — Utilise le BIOS/UEFI normalisé NVD (bios_version_id).

Source vendor

  • os_fk — Vendor NVD extrait de l'OS normalisé. Pour PC, serveurs, laptops.
  • fw_fk — Vendor NVD extrait du firmware normalisé. Pour équipements réseau.
  • materiel — Vendor NVD du fabricant matériel de l'asset. Pour NAS, caméras, lecteurs.
  • detection_auto — Essaie os_fk, fw_fk, OS textuel, puis vendor matériel dans l'ordre.

Utilisation

  • Cliquez sur une ligne pour l'éditer directement.
  • La colonne Assets indique combien d'assets utilisent ce type — un type avec assets ne peut pas être supprimé.
  • Les modifications prennent effet au prochain lancement de la corrélation.
